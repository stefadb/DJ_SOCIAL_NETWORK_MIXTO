API DEGLI ALBUM:

2) - PER OTTENERE TUTTI GLI ALBUM DI UN ARTISTA - OK

https://api.deezer.com/artist/5359276/albums

Restituisce tutti gli album dell'artista e il primo genere, quello più importante.

Upsert: album
Caricamento foto: album

Usa sempre limit e index per la paginazione

API DEI BRANI:

3) - PER OTTENERE TUTTI I BRANI CHE CORRISPONDONO A UNA PAROLA/FRASE DI RICERCA (query) - OK

https://api.deezer.com/search/track?q=Parola%20Chiave

Per ogni traccia restituita, restituisce tutte le info necessarie sulla traccia, tutte le informazioni necessarie sull'artista PRINCIPALE (non tutti gli artisti) e tutte le informazioni necessarie sull'album (ad eccezione del genere)

L'artista è nell'oggetto artist di ogni brano
L'album è nell'oggetto album di ogni brano

Upsert: artista, album, brano
Caricamento foto: artista, album
Associazioni: brano_artista (TODO: per ogni brano, aggiunge, se non esiste già, l'associazione tra il brano e l'artista principale)

Usa sempre limit e index per la paginazione

11) - PER OTTENERE GLI ARTISTI DI UN ALBUM DATO IL SUO ID - OK

Gli artisti si ottengono solo ottenendo le tracce dell'album e unendo tutti gli artisti di ogni traccia

Questa API dovrà ottenere tutti gli artisti dell'album, restituirli e ovviamente upsertarli sul db con le loro foto

Crea un array contributors, inizialmente vuoto, che andrà riempito

API 1: https://api.deezer.com/album/221700932

API 2: Per ogni id traccia trovato: https://api.deezer.com/track/1334599202

A ogni chiamata della API 2, ottieni i contributors, cioè la lista completa degli artisti della traccia. 

Per ogni contributor trovato, fai l'upsert dell'artista e il caricamento foto, poi lo aggiungi all'array contributors se ancora manca, rispettando il formato sul db

Alla fine, restituisci l'array contributors al frontend

Chiamate API complessive: 1 + N, dove N è il numero di tracce dell'album
